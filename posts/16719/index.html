类型大小

几乎所有C语言中的基本数据类型的大小都是依赖于编译器和运行时环境的。这意味着它们在不同的操作系统、硬件架构、甚至不同的编译器配置下可能会有不同的大小。这种设计使得C语言能够在广泛的硬件上运行，从嵌入式系统到高性能计算服务器，但也意味着编写可移植代码时需要特别注意数据类型的选择。

C语言基本数据类型的环境依赖性：

以下是一些C语言中常见的基本数据类型，以及它们大小的环境依赖性：

  • char：char类型用来存储单个字符。按照标准，char的大小是1个字节。不过，字节的大小并不是在所有系统上都是8位；虽然如今几乎所有现代系统上都是8位。
  • int：int的大小旨在反映机器上自然的整数大小，以便提供最佳的性能。它通常是16位或32位，但在某些系统上可能是64位。
  • short：short int（简写为short）至少为16位。
  • long：long int（简写为long）至少为32位，在许多64位系统上是64位。
  • long long：C99标准添加的long long int（简写为long long）类型至少为64位。
  • 浮点类型（float、double、long double）：这些类型用于表示浮点数，它们的大小和精度也因环境而异。通常，float是32位，double是64位，而long double的大小至少和double一样，但具体大小取决于实现。
  • 指针类型：指针类型的大小依赖于系统的地址空间。例如，在32位系统上，指针通常是32位的，在64位系统上，指针通常是64位的。

为了确保程序的可移植性，C语言提供了sizeof运算符，允许程序在运行时查询数据类型的大小。此外，C标准库中的<stdint.h>头文件定义了一组整数类型，它们具有明确的大小（如int16_t、int32_t、int64_t等），这些类型有助于编写在不同环境下表现一致的程序。

64位机器上，C语言各个类型的大小

在64位机器上，C语言中基本数据类型的大小可能会根据编译器和操作系统的不同有所差异，但通常遵循以下规则：

  • char: 1字节。char 类型保证为1字节，用于存储单个字符。
  • short: 通常是2字节。
  • int: 通常是4字节。在许多64位系统上，int 保持为32位以保持与32位系统的兼容性。
  • long: 在64位系统上，long 通常是8字节（64位）。这是与32位系统上的long（通常为4字节）的主要区别之一。
  • long long: 至少8字节。long long 类型保证至少64位。
  • float: 通常是4字节。float 类型用于单精度浮点数。
  • double: 通常是8字节。double 类型用于双精度浮点数。
  • long double: 大小可能变化，但通常是8字节或更大，具体取决于编译器和系统。long double 用于提供比double更高的精度和范围。
  • 指针: 在64位系统上，所有类型的指针（无论是指向char、int、float还是任何其他类型的指针）都是8字节。指针大小增加是为了能够寻址更大的内存空间。

请注意，这些大小是典型值，实际上可能会有所不同。特别是long和long double类型的大小可能会根据不同的编译器和操作系统而变化。为了精确地了解特定环境下每种类型的大小，可以使用sizeof运算符进行检查，例如sizeof(int)。这会给出在当前编译环境下int类型的确切字节数。

NULL

如何理解c语言中的NULL，‘\0’，0

ChatGPT:

在C语言中，NULL、'\0'和0有着不同的含义和用途，尽管它们在某些情况下看起来可能相似。理解它们之间的区别对于编写清晰、正确的C程序至关重要。

'\0'

  • '\0'表示的是空字符（null character），用于标记C字符串的结束。它是一个字符字面量，其ASCII码值为0。在内存中，'\0'占用一个字节，并且其字节值为0。
  • 用途：主要用于字符串的结束标志。当字符串处理函数（如strlen或strcpy）操作字符串时，它们依赖于'\0'来确定字符串的结束位置。

0

  • 0是一个整数字面量，代表数值零。在不同的上下文中，0可以有不同的含义，但本质上，它表示数值上的零。
  • 用途：
    • 作为整数使用，表示数值0。
    • 在指针上下文中，0被隐式地视为NULL，用于表示空指针。然而，使用NULL而不是0来表示空指针是更好的做法，因为这样做可以提高代码的清晰度和可读性。

NULL

  • NULL是一个宏，用于表示空指针常量。在C标准库中，通常通过#define NULL ((void*)0)来定义。这意味着NULL代表一个指向无类型（void类型）的零地址的指针。
  • 用途：主要用于指针变量，表示指针不指向内存中的任何有效地址。使用NULL来初始化或赋值给指针变量可以清晰地表示该指针目前没有指向任何有效的对象或数据。

总结

  • '\0'是一个空字符，用于字符串的结束。
  • 0是数值零，可以表示整数零或在指针上下文中隐式表示空指针。
  • NULL是一个宏，用于表示空指针，使代码更清晰地表达出指针的“空”状态。

在编程时选择正确的表示形式对于保证代码的意图清晰和准确非常重要。尤其是在涉及指针和字符串操作时，正确使用这些概念可以避免许多常见的错误。

指针的算数操作

在C语言中，指针的算术操作有以下规则：

  1. 指针加上或减去一个整数：当你对一个指针加上一个整数n时，结果是一个新的指针，它指向原来的指针所指向的元素之后的第n个元素。类似地，当你从一个指针减去一个整数时，结果是一个新的指针，它指向原来的指针所指向的元素之前的第n个元素。这种操作依赖于指针所指向的类型的大小。

    为什么p + 3能正好跨越3个int?

    这个行为是由于C语言中指针算术的设计原则所决定的。当你对一个指针执行算术操作时，加上或减去的整数值会根据指针所指向的数据类型的大小进行缩放。这意味着，当你对指针p加上3（p = p + 3;），实际上你不是简单地在p的数值上加3，而是让p前进三个由p所指类型的数据大小决定的内存单位。

    在你的例子中，p是一个指向int的指针。假设在你的系统中，int类型占用4个字节的内存空间。因此，当你写p + 3时，实际上是将p的值增加了3 * sizeof(int)字节。这就意味着p会前进到内存中的下一个位置，该位置恰好是数组中第四个元素的位置。

    这个行为使得指针算术非常适用于遍历数组或者访问数组中的特定元素，因为你可以简单地通过加上或减去一个整数来移动到数组中的不同位置。这也保证了不同数据类型的数组可以使用统一的方式来进行指针操作，而不用担心数据类型大小的问题。

  2. 两个指针之间的减法：当你从一个指针减去另一个指针时，结果是这两个指针之间的元素数量。这两个指针通常指向同一个数组或数据结构的不同部分。这种操作的结果是ptrdiff_t类型，这是一个有符号整型，足以存储两个指针之间的差异。

C语言中的空类型指针（void pointer）是一种特殊的指针类型，它可以指向任何类型的数据。空类型指针通常被称为“泛型指针”。在C语言中，void 关键字用于表示没有具体类型的情况，因此，一个 void 类型的指针就是一个没有指定所指向的数据类型的指针。

空类型指针的特点：

  • 泛型性：因为它们没有指定数据类型，所以 void 指针可以被用来指向任何类型的数据。这使得 void 指针在处理不同数据类型时非常灵活。

C语言引用和C++引用

C++中的“函数引用”与C语言中讨论的“引用（&）”和“解引用（*）”操作有着本质的不同。C++的引用是对C语言中指针概念的一种扩展和封装，旨在提供一种更安全、更方便的方式来操作变量的别名，而不需要像操作指针那样直接处理内存地址。以下是C++引用和C语言中指针操作的主要区别：

C++引用

  1. 别名：C++中的引用就是变量的别名。创建引用时，它就成了被引用变量的另一个名字。
  2. 初始化：引用在创建时必须被初始化，并且一旦被初始化指向一个变量，就不能改变指向其他变量。引用的这个特性保证了引用的稳定性和安全性。
  3. 使用简便：使用引用就像使用普通变量一样，不需要特殊的操作符来访问引用的值。
  4. 类型安全：C++引用更加类型安全，编译器会进行类型检查，减少了出错的可能。

C语言中的指针操作

  1. 内存地址：指针本质上是存储另一个变量的内存地址的变量。通过指针，可以间接访问和操作该地址处的数据。
  2. 灵活性：指针可以在任何时候改变指向，指向另一个变量或者动态分配的内存。
  3. 操作复杂：使用指针访问变量的值需要使用解引用操作符（*），而且需要正确管理内存（如分配和释放动态内存）。
  4. 潜在危险：如果不正确使用，指针可能导致程序错误，如空指针解引用、野指针等。

结论

C++中的引用相比于C语言的指针操作，提供了一种更为安全、简洁的方式来引用另一个变量。它封装了指针的一些复杂性，使得代码更易读、易写、易维护。然而，指针因其提供的灵活性和直接的内存操作能力，在很多场景下仍然是必不可少的。

这个特性在需要将宏参数直接转换为字符串的场合非常有用，例如，在生成错误消息、打印日志、或者创建格式化的文本时。字符串化操作符#使得预处理器在编译之前就完成了这种转换，从而避免了在运行时进行字符串操作，这可以提高程序的效率。

③#define N 2+2

void main()

{

int a=N*N;

printf(“%d”,a);

}

出现问题：在此程序中存在着宏定义命令，宏N代表的字符串是2+2，在程序中有对宏N的使用，一般同学在读该程序时，容易产生的问题是先求解N为 2＋2＝4，然后在程序中计算a时使用乘法，即NN=44=16,其实该题的结果为8，为什么结果有这么大的偏差?

问题解析：如1节所述，宏展开是在预处理阶段完成的，这个阶段把替换文本只是看作一个字符串，并不会有任何的计算发生，在展开时是在宏N出现的地方 只是简单地使用串2＋2来代替N，并不会增添任何的符号，所以对该程序展开后的结果是a=2+2*2+2，计算后=8，这就是宏替换的实质，如何写程序才 能完成结果为16的运算呢？

解决办法：将宏定义写成如下形式

#define N (2+2)

这样就可替换成(2+2)*(2+2)=16

④#define area(x) x*x

void main()

{

int y=area(2+2);

printf(“%d”,y);

}

按理说给的参数是2+2，所得的结果应该为44=16，但是错了，因为该程序的实际结果为8，仍然是没能遵循纯粹的简单替换的规则，又是先计算再替换 了，在这道程序里，2+2即为area宏中的参数，应该由它来替换宏定义中的x，即替换成2+22+2=8了。那如果遵循(1)中的解决办法，把2+2 括起来，即把宏体中的x括起来，是否可以呢？#define area(x) (x)(x)，对于area(2+2)，替换为(2+2)(2+2)=16，可以解决，但是对于area(2+2)/area(2+2)又会怎么样呢，有的学生一看到这道题马上给出结果，因为分子分母一样，又错了，还是忘了遵循先替换再计算的规则了，这道题替换后会变为(2+2)(2+2)/(2+2)(2+2)即44/44按照乘除运算规则，结果为16/44=44=16，那应该怎么呢？解决方法是在整个 宏体上再加一个括号，即#definearea(x) ((x)*(x))，不要觉得这没必要，没有它，是不行的。

要想能够真正使用好宏定义，那么在读别人的程序时，一定要记住先将程序中对宏的使用全部替换成它所代表的字符串，不要自作主张地添加任何其他符号，完全展开后再进行相应的计算，就不会写错运行结果。如果是自己编程使用宏替换，则在使用简单宏定义时，当字符串中不只一个符号时，加上括号表现出优先级，如果是 带参数的宏定义，则要给宏体中的每个参数加上括号，并在整个宏体上再加一个括号。

① 预处理功能：

（1）文件包含：可以把源程序中的#define扩展为文件正文，即把包含的.h文件找到并展开到#include所在处。

（2）条件编译：预处理器根据#if和#ifdef等编译命令及其后的条件，把源程序中的某些部分包含进来或排除在外，通常把排除在外的语句转换成空行。

（3）宏展开：预处理器将源程序文件中出现的对宏的引用展开成相应的宏定义，经过预处理器处理的源程序与之前的源程序有所不同，在这个阶段所进行的工作只是纯粹的替换和展开，没有任何计算功能。

②使用带参数的宏定义可完成函数调用的功能，又能减少系统开销，提高运行效率。

正如C语言中所讲，函数的使用可以使程序更加模块化，便于组织，而且可重复利用，但在发生函数调用时，需要保留调用函数的现场，以便子函数执行结束后能返回继续执行，同样在子函数执行完后要恢复调用函数的现场，这都需要一定的时间，如果子函数执行的操作比较多，这种转换时间开销可以忽略，但如果子函数完成的功能比较少，甚至只完成一点操作，如一个乘法语句的操作，则这部分转换开销就相对较大了，但使用带参数的宏定义就不会出现这个问题，因为它是在预处理阶段即进行了宏展开，在执行时不需要转换，即在当地执行。宏定义可完成简单的操作，但复杂的操作还是要由函数调用来完成，而且宏定义所占用的目标代码空间相对较大。所以在使用时要依据具体情况来决定是否使用宏定义。

编译相关

从C语言源代码到最终的可执行程序，整个过程经历了几个关键步骤。这个转换过程通常由编译器（如GCC）和其他工具链组件自动完成。下面是这一过程的简化概述：

1. 预处理（Preprocessing）

  • 目的：处理源代码文件中的预处理指令，如#include（插入头文件），#define（定义宏），以及#if、#ifdef、#ifndef等条件编译指令。
  • 结果：生成一个扩展的源代码文件，其中包含了所有通过预处理指令包含或定义的内容，不再含有任何预处理指令。

2. 编译（Compilation）

  • 目的：将预处理后的源代码转换成中间表示或汇编语言。
  • 结果：生成汇编语言文件。这一步涉及源代码的语法和语义分析，错误检查，以及优化代码。

3. 汇编（Assembly）

  • 目的：将汇编语言代码转换为机器语言指令，即生成目标代码。
  • 结果：生成目标文件（通常是.o或.obj文件）。汇编器将汇编语言指令转换成机器码。

4. 链接（Linking）

  • 目的：将一个或多个目标文件与库文件一起合并，解决符号引用/定义之间的依赖，生成最终的可执行文件。
  • 结果：生成可执行程序（在Windows上是.exe文件，在Unix-like系统上通常没有扩展名）。

这个过程可以被分解为更细的步骤，具体取决于编译器和操作系统，但上述给出的是一个通用和高层次的概述。每个步骤都可能引入不同的优化和处理，以提高代码的性能或减小可执行文件的大小。此外，现代开发环境和编译器可能会提供额外的功能，如跨编译（为不同于编译它的系统构建程序）、增量编译（只重新编译改动的部分）等。
